# Task: fft-frequency-tab

## Overview
为桌面工作台增加一个独立的 FFT 频谱分析 Tab，用于展示当前数据波形的频率分量，帮助快速识别主频、谐波与噪声分布。

## Requirements
- 在 `Workbench` 中新增独立 Tab（与现有波形/逻辑分析页面并列），用于频域分析展示。
- 频谱数据来源于当前实时采样数据，确保与现有波形窗口对齐（同一段数据窗口）。
- 在 UI 中显示至少一个主频谱图（频率-幅值），并支持基本可读性信息（主峰频率/幅值）。
- 与现有状态流和桥接模型兼容：若后端扩展分析字段，需同步更新 Tauri/前端类型契约。
- 保留现有时域分析与假流调试路径，不破坏当前波形页与串口采集流程。
- 为新增 FFT/频谱链路补充测试（至少覆盖核心分析或载荷映射路径）。

## Acceptance Criteria
- [ ] Workbench 中可切换到新的 FFT 频谱独立 Tab。
- [ ] Tab 内可稳定显示当前波形对应窗口的频率分量图。
- [ ] 页面可展示并更新主峰频率（或等价核心频域指标）。
- [ ] 未连接设备或样本不足时，页面有明确空态/降级展示，不报错崩溃。
- [ ] 现有波形、变量面板、逻辑分析页功能不回归。
- [ ] 相关测试通过（Rust `cargo test` 与前端 `npm --prefix ui run build`）。

## Technical Notes
- 当前代码主要是时域分析，尚无现成 FFT 输出结构；需要明确频谱结果落点（后端分析模型或前端计算层）。
- FFT 计算依赖稳定采样窗口和采样率假设；需明确使用的窗口长度、采样率来源和不足样本时行为。
- 若选择后端计算，需要扩展 `src/analysis/model.rs`、`src/message.rs` 与 `src-tauri/src/model.rs` 的契约映射。
- 若选择前端计算，需要确认 `ui/src/store/appStore.ts` 中可获取足够长度的连续样本并控制计算开销。
- 独立 Tab 形态可复用 `Workbench` 与 `LogicAnalyzerPage` 的布局模式，降低耦合风险。

## Out of Scope
- 不包含高级频谱参数面板（窗函数切换、重叠率、平均算法等）的大规模配置化。
- 不包含离线频谱历史持久化与导出。
- 不包含多通道同步频谱矩阵/瀑布图等高级可视化。
- 不修改设备端协议或串口底层传输机制。
